# Logo & brand mark — notes

Status: `current`. Source of truth for the mark geometry is `lib/brand/mark.ts`;
all assets are generated by `scripts/gen-brand.mts` (`pnpm gen:brand`).

## The mark

A constructed capital **P** whose bowl-counter holds a teal **command prompt** — a
`>` caret followed by a cursor dot. The letterform carries the name ("Prompt"); the
caret turns the counter into _the place where you type_, and the cursor dot is the
entry it produces. It reads as one engineered glyph, not a letter dropped in a tile.

```
violet P  +  teal ">"  +  teal cursor dot   →   "PromptFlow"
(the name)   (the prompt)  (flow / the stored entry)
```

### Why it fits

PromptFlow is a keyboard-native "dark productivity cockpit" — a private instrument
for storing, versioning and reusing prompts. The mark encodes the product's own
vocabulary instead of a generic category cue:

- **Prompt** → the `>` command prompt the whole app is built around (⌘K, capture, copy).
- **Flow / library** → the cursor dot: the prompt flows forward and lands as a stored,
  versioned entry. Reinforced by the **Prompt**(medium) / **Flow**(semibold) weight-step
  in the wordmark.
- **Cockpit / engineered** → a precise, constructed letterform with a single restrained
  violet gradient and one teal accent. Confident and quiet, not loud.

It deliberately avoids every project anti-reference (`docs/PRODUCT.md`): no lightning
bolt, no AI sparkle/star, no neon-on-black, no plain-initial-in-a-circle.

### Alternatives explored

Six directions were designed, rendered at 16–512px, and scored by an independent
3-judge panel (art-director / product-brand / icon-engineer). Summed scores:

| Direction | Idea | Score |
| --- | --- | --- |
| **p-monogram** ✅ | constructed P + caret in the counter | **244** |
| caret-flow | `>` caret releasing a flow line to a node | 216 |
| stack-flow | versioned stack threaded by a flow line | 201 |
| aperture | cockpit instrument iris around a caret | 197 |
| bracket-cursor | code brackets + a striding cursor | 180 |
| flow-graph | routed node/flow graph | 177 |

`p-monogram` won as the only mark that is simultaneously distinctive, premium, **and**
robust at favicon scale (the line-based marks — caret-flow especially — collapse into a
teal blob at 16px and lose the prompt reading). Two refinements were grafted from the
runners-up: the teal **cursor dot** (flow/stored-entry meaning, from caret-flow) and a
hairline of dark counter so teal never touches violet (optical separation, from aperture).

## Animation

The animated mark plays **once on mount, then rests** — a micro-interaction, not a loop:

1. the teal caret is "typed" in (`stroke-dashoffset` draw, ~0.55s);
2. the cursor dot lands (opacity + scale pop, ~0.2s);
3. one soft cursor pulse settles (opacity 1 → 0.82 → 1), then full rest.

The violet **P is fully static and complete from frame 0** — the animation is pure
delight and never load-bearing.

**Why it's lightweight & safe:** CSS-only (`stroke-dashoffset`, `opacity`, `transform`
— all GPU-cheap), no JS, no SMIL, no filters/blur, no perpetual motion. It animates only
the small teal elements, never layout, so there is **no layout shift** and the navbar
stays readable. The whole thing is wrapped in `@media (prefers-reduced-motion:
no-preference)`; under reduced motion (or no CSS at all) the fully drawn static mark
renders immediately.

## Usage

**Animated** — high-visibility brand moments only: the sidebar header (`BrandWordmark
animated`) and the unlock welcome hero (`BrandLogoAnimated`).

**Static** (`BrandLogo`) — everywhere else: favicons, app icons, OG image, footers,
repeated/dense dashboard UI, mobile. The static mark is strong on its own and must be
used anywhere performance or visual calm matters more than delight.

## Color & background

- The **P** is a single 2-stop violet gradient (`#a78bfa → #7c3aed`); a flat `#8b5cf6`
  is the fallback for favicons / mono contexts.
- The **counter is open negative space**, so the teal accent sits on whatever surface is
  behind the mark. On dark it's teal-400 `#2dd4bf`; on light it switches to teal-500
  `#14b8a6` (AA-legible on near-white) via the `--pf-accent` token. Standalone files use
  `mark-light.svg` for light surfaces.
- App-icon tiles use the elevated cockpit base `#1d1a28` with a faint violet glow.

## Performance / reduced-motion

- No animation library, no runtime motion cost beyond a one-shot CSS reveal on the small
  teal elements. The mark SSRs in the initial HTML → no flash, no layout shift.
- `prefers-reduced-motion: reduce` → static mark, no exceptions. Favicons/app icons/OG
  are static by definition.
